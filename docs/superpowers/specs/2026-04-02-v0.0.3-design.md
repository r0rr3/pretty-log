# pretty-log v0.0.3 设计文档

**日期**：2026-04-02
**版本**：0.0.3
**基于**：v0.0.2

---

## 一、概述

v0.0.3 包含两项独立改进：

1. **普通模式视觉优化** — 改善 level/message/extras 三层视觉层次
2. **表格模式（`-t`）** — 基于 ratatui 的交互式 TUI 表格视图

---

## 二、普通模式颜色层次变更

### 改动目标

当前 level 与 message 视觉粘连，extras 字段与 message 区分度不足。

### 新颜色规则

| 字段 | 旧行为 | 新行为 |
|------|--------|--------|
| `level` | 纯文字 + 前景色 | ANSI 背景色徽章（前景 + 背景） |
| `message` | `bold().white()` | `white()`（ERROR 时 `red()`） |
| `extras` key | `yellow()` | `dark_gray()` |
| `extras` value | 白色 | `dark_gray()` |
| `trace` / `caller` | 各自独立色 | 统一 `dark_gray()` |

### Level 徽章配色

| Level | 前景 | 背景 |
|-------|------|------|
| ERROR | bright_red | on_dark_red |
| WARN | bright_yellow | on_dark_yellow |
| INFO | bright_green | on_dark_green |
| DEBUG | bright_blue | on_dark_blue |
| TRACE | dark_gray | on_black |

### 输出示例

```
10:23:45 [INFO]  server started on port 8080  host=localhost port=8080
10:23:46 [WARN]  slow query detected  duration_ms=1500 query=SELECT *
10:23:47 [ERROR] connection refused  trace=abc-123 caller=main.rs:42
```

（`[INFO]` 等为带背景色的 ANSI 徽章，extras 整体为暗灰色）

## 三、表格模式（`-t` / `--table`）

### 激活方式

```bash
tail -f app.log | pretty -t
tail -f app.log | pretty -t -x      # 同时开启 extras 展示
cat app.log | pretty -t
```

### 技术栈

- `ratatui 0.29` — TUI 布局与渲染
- `crossterm 0.28` — 跨平台终端控制、键盘/鼠标事件

仅在 `-t` 激活时引入事件循环，普通模式代码路径不变。

### 列定义

默认列顺序（可通过 `table.columns` 配置调整）：

```
time(8) │ level(7) │ message(自适应) │ service(12) │ label(10) │ trace_id(14)
```

- 固定列宽在启动时根据终端宽度自适应比例分配，运行时不变
- `message` 列独占剩余所有宽度
- 超出列宽的内容截断显示，选中后 Enter 展开

### 交互操作

| 操作 | 行为 |
|------|------|
| `↑` / `↓` | 移动光标行 |
| 鼠标滚轮 | 上下滚动 |
| `Enter` | 展开/收起当前行 |
| `End` / `G` | 跳到最新行，清除新日志提示 |
| `Space` | 暂停/继续自动跟随最新行 |
| `q` / `Ctrl-C` | 退出 |

### 实时流行为

向上滚动时：
- 继续接收并缓冲新日志，视图不跳动
- 状态栏显示 `↓ N new`
- 按 `End` 跳回最新行并清除提示

### 展开行（Enter）内容

展开面板显示：
1. 完整 message（不截断）
2. extras 字段（受 `show_extras_in_detail` 控制，默认 `false`）
3. 续行内容（堆栈跟踪等）

### 状态栏

```
↑↓/滚轮 滚动  ·  Enter 展开  ·  Space 暂停  ·  q 退出        row 42/156  ·  ● LIVE
```

## 四、配置变更

`config.rs` 新增 `TableConfig`：

```yaml
table:
  columns: [time, level, message, service, label, trace_id]
  show_extras_in_detail: false
```

- `columns`：列显示顺序，可省略任意列
- `show_extras_in_detail`：展开行时是否显示 extras 字段

---

## 五、CLI 变更

`main.rs` 新增两个参数：

| 参数 | 说明 |
|------|------|
| `-t`, `--table` | 开启表格模式 |
| `-x`, `--extras` | 展开行时显示 extras（覆盖 config `show_extras_in_detail`） |

`-x` 仅在 `-t` 模式下生效，普通模式忽略。
